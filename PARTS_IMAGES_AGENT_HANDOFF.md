# Промпт для агента: підключити локальні схеми запчастин до вебсторінки

Працюй у репозиторії `xainse/mini-cooper-r50-db`.

## Що вже підготовлено

- Завантажено всі **639** зображень схем із переліку `PARTS_IMAGES_DOWNLOAD_TASK.md`.
- Файли лежать у папці `assets/parts/`.
- Усі файли є валідними JPEG-зображеннями.
- Назва кожного файла утворена з `diagId` за точним правилом:

  ```text
  assets/parts/diag-<diagId>.jpg
  ```

- Приклади:

  ```text
  diagId: 01_0164 -> assets/parts/diag-01_0164.jpg
  diagId: 11_1118 -> assets/parts/diag-11_1118.jpg
  diagId: 83_0336 -> assets/parts/diag-83_0336.jpg
  ```

- Точний повний список усіх 639 `diagId`, назв схем і відповідних шляхів міститься у `PARTS_IMAGES_DOWNLOAD_TASK.md`, у полі `Файл для архіву` кожного пункту.
- Фізично є 639 файлів, але 591 унікальний вміст: деякі різні схеми RealOEM навмисно використовують однакову картинку. Не видаляй такі дублікати, бо кожен `diagId` повинен мати власний передбачуваний локальний шлях.
- Сукупний розмір папки — приблизно 34 МБ.

## Завдання

Підключи локальні зображення до сторінки каталогу запчастин.

1. Відкрий `data/parts-catalog.json` і знайди масив об'єктів схем, у яких є поле `diagId`.
2. Для кожної з 639 схем додай або онови поле:

   ```json
   "imageLarge": "assets/parts/diag-<diagId>.jpg"
   ```

   Підставляй значення `diagId` без змін, включно з підкресленням.
3. Не додавай посилань на RealOEM або інші зовнішні каталоги на публічну вебсторінку.
4. Якщо після підключення картинки поле `imageStatusUk` із текстом про відсутню локальну схему більше не потрібне, видали його або зміни логіку так, щоб воно не показувалося для наявних картинок.
5. Збережи наявні OEM-таблиці, переклади, структуру груп і порядок схем без змін.
6. Перевір `script.js`: зараз `renderDiagrams()` уже бере зображення через `diagram.imageLarge || diagram.image`, тому бажаний контракт — заповнити `imageLarge` у JSON без дублювання URL у HTML.
7. Перевір результат на `parts.html` через локальний HTTP-сервер:
   - зображення схем відображаються;
   - пошук і фільтрація продовжують працювати;
   - у консолі немає 404 для `assets/parts/diag-*.jpg`;
   - картки без OEM-рядків також показують відповідну схему;
   - верстка коректна на вузькому й широкому екрані.

## Обов'язкова перевірка повноти

Перед завершенням програмно звір:

- у `data/parts-catalog.json` рівно 639 схем із `diagId`;
- кожна схема має шлях `assets/parts/diag-<diagId>.jpg`;
- кожен такий файл існує на диску;
- немає зовнішніх URL зображень у даних, які використовує публічна сторінка;
- не залишилося 404 або повідомлень «Схему ще не перенесено локально» для цих 639 схем.

Після змін дай короткий звіт: які файли змінено, скільки схем підключено, результат перевірки 639/639 і як саме перевірено сторінку.
